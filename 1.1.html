Visión General del Proceso

Conceptos de la Ingeniería de Software.

Enfoque sistemático, disciplinado y cuantificable del desarrollo, operación y mantenimiento de software. Nato, 1968

La construcción de múltiples versiones de un software llevada a cabo por múltiples personas. Parnas, 1978

Construcción de software de una envergadura o complejidad tales que debe ser construido por equipos de ingenieros. Ghezzi 1991

La ingeniería tradicional es altamente especializada y se basa en colecciones de diseños estándar o normalizados. Jackson 1998



¿Qué hace el Ingeniero de Software?

  • Construye máquinas de software: Un ingeniero de software no construye el hardware, sino el comportamiento y las propiedades que lo haran útil para algo específico.

  • Desarrolla o documenta descripciones de un software: La actividad central del desarrollo de softare es la descripción. Cualquier desarrollo de software requiere muchas descripciones.

Descripciones fundamentales.

Requerimientos del usuario (R)
En esta etapa, se recopilan y documentan las necesidades y expectativas del usuario para el software. Estos requisitos se expresan en términos de funciones, características y restricciones que el software debe cumplir.

Diseño de la estructura del programa (D)
En esta etapa, se crea una arquitectura o estructura lógica para el software. Se definen los componentes del sistema, sus interacciones y se establece cómo se organizarán para cumplir con los requisitos del usuario.

Especificación funcional del programa (S)
En esta etapa, se detalla cómo funcionará el software desde una perspectiva funcional. Se describen las operaciones, algoritmos y reglas que el software seguirá para llevar a cabo sus tareas.

Programa (P)
En esta etapa, los programadores escriben el código fuente del software siguiendo las especificaciones y el diseño previamente definidos. Se implementan algoritmos, se crean funciones y se desarrollan las características del software.
